# МедТест

PWA-тренажёр для подготовки к медицинским экзаменам с системой платного доступа по коду активации.

---

## Возможности

### Типы вопросов

| Тип | Описание |
|---|---|
| `q/a` | Свободный ответ — вопрос и текстовый ответ |
| `single` | Одиночный выбор из вариантов |
| `multi` | Множественный выбор (несколько правильных) |
| `matching` | Сопоставление левого и правого столбцов |
| `fill_each` | Заполнение пропусков в тексте |
| `sequence` | Расстановка элементов в правильном порядке |
| `assertion` | Утверждение-обоснование (5 фиксированных вариантов) |

### Режимы работы

- **Тёмная и светлая тема** — переключение сохраняется между сессиями
- **Перемешивание** — случайный порядок вопросов и вариантов ответов
- **Режим «Ошибочные»** — повторная проработка только тех вопросов, на которых были ошибки
- **Выбор диапазона** — работа с произвольным подмножеством вопросов темы
- **Экран завершения** — результат в процентах, количество ошибок, время сессии
- **Клавиатурные сокращения** — пробел/стрелки для управления без мыши

### Авторизация и доступ

- Регистрация и вход по email + пароль через Supabase Auth
- Сброс пароля по email
- Доступ к тренажёру открывается после ввода **кода активации**; доступ ограничен по сроку
- Продление доступа — активация нового кода из раздела «Профиль»
- **Бессрочный доступ** — особый флаг `unlimited_access` в профиле; администратор может выдать или отозвать его любому пользователю

### Профиль пользователя

- Статус доступа: срок действия или бессрочный
- Поле для ввода кода активации прямо в профиле
- Для администратора — блок управления доступом других пользователей (выдать / отозвать по email)

### «Предложить вопросы»

- Раздел доступен любому авторизованному пользователю с активным доступом
- Загрузка файла с вопросами (до 25 МБ) в Supabase Storage
- После загрузки запись попадает в таблицу `question_submissions`
- Edge Function `notify-submission` отправляет администратору уведомление в Telegram: сообщение с деталями заявки и сам файл как вложение

### PWA

- Устанавливается на телефон как приложение (через браузерный диалог «Добавить на экран»)
- Работает офлайн после первой загрузки — Service Worker кэширует все темы и ресурсы

## Технологии

| Слой | Что используется |
|---|---|
| Frontend | Чистый HTML + CSS + JS, без фреймворков и сборщиков |
| Иконки | [Lucide](https://lucide.dev/) через CDN (UMD) |
| Auth + БД | [Supabase](https://supabase.com/) — Auth (email/пароль, PKCE), Postgres (профили, коды доступа, заявки) |
| Хранилище файлов | Supabase Storage (bucket `submissions`) |
| Серверная логика | Supabase Edge Functions (Deno) |
| Уведомления | Telegram Bot API |
| Хостинг | GitHub Pages |

## Запуск локально

```bash
npx serve .
```

Или через Python:

```bash
python -m http.server 8080
```

Затем откройте `http://localhost:8080` в браузере.

> **Важно:** авторизация через Supabase работает только по `https://` или `http://localhost`.
> Открытие `index.html` через `file://` даёт базовый UI, но вход/регистрация работать не будут.

## Как добавить новую тему

1. Создайте файл `topics/<название>.json` с вопросами.
2. Добавьте запись в `topics_index.json` (2-пробельный отступ, порядок полей `title` → `file`):
   ```json
   { "title": "Название темы", "file": "<название>.json" }
   ```
3. Проверьте корректность файла перед коммитом:
   ```bash
   node validate-topics.js
   ```
4. Тема появится в выпадающем списке автоматически.

Все поддерживаемые форматы вопросов, обязательные поля, допустимые значения `correct` и примеры — в [CLAUDE.md](CLAUDE.md).

## Деплой

Проект деплоится на **GitHub Pages** из ветки `master`, корневая папка `/`.

Настройка: репозиторий → Settings → Pages → Branch: `master`, folder: `/ (root)`.

Манифест и Service Worker используют относительные пути (`"./"`, `"start_url": "."`), поэтому приложение корректно работает под любым подпутём, например `username.github.io/Test3/`.
