# МедТест

PWA-тренажёр для подготовки к медицинским экзаменам с системой платного доступа по коду активации.

---

## Возможности

- **Несколько типов вопросов:** свободный ответ (q/a), одиночный выбор, множественный выбор, сопоставление, заполнение пропусков, последовательность, утверждение-обоснование
- **Тёмная и светлая тема** с сохранением выбора
- **Перемешивание** — случайный порядок вопросов и вариантов ответов
- **Режим «Ошибочные → новый прогон»** — повторная проработка вопросов, на которых были ошибки
- **Авторизация по email** (регистрация, подтверждение, вход) через Supabase Auth
- **Коды активации** — доступ открывается после ввода кода, ограничен по сроку
- **PWA** — устанавливается на телефон как приложение, работает офлайн после первой загрузки

---

## Технологии

| Слой | Что используется |
|---|---|
| Frontend | Чистый HTML + CSS + JS, без фреймворков и сборщиков |
| Иконки | [Lucide](https://lucide.dev/) через CDN |
| Бэкенд / Auth | [Supabase](https://supabase.com/) — Auth (email) + Postgres (профили, коды доступа) |
| Хостинг | GitHub Pages |

---

## Структура проекта

| Файл / папка | Назначение |
|---|---|
| `index.html` | Разметка всех экранов: загрузка, вход, активация, тренажёр |
| `style.css` | Все стили; тёмная тема по умолчанию, светлая через `.light` на `<body>` |
| `app.js` | Логика тренажёра: загрузка тем, отображение вопросов, проверка ответов |
| `auth.js` | Supabase-клиент, функции входа/регистрации/выхода, проверка доступа |
| `service-worker.js` | PWA-кэш для офлайн-работы |
| `manifest.json` | Манифест PWA (имя, иконки, цвета) |
| `topics_index.json` | Список доступных тем (заголовок + имя файла) |
| `topics/` | JSON-файлы с вопросами по темам |

## Запуск локально

```bash
npx serve .
```

Или через Python:

```bash
python -m http.server 8080
```

Затем откройте `http://localhost:8080` в браузере.

> **Важно:** авторизация через Supabase работает только по `https://` или `http://localhost`.
> Открытие `index.html` напрямую через `file://` даст базовый UI, но вход/регистрация работать не будут.

## Как добавить новую тему

1. Создайте файл `topics/<название>.json` с вопросами в нужном формате.
2. Добавьте запись в `topics_index.json`:
   ```json
   { "title": "Название темы", "file": "<название>.json" }
   ```
3. Тема появится в выпадающем списке автоматически.

Поддерживаемые форматы вопросов и примеры структуры JSON описаны в [CLAUDE.md](CLAUDE.md).
